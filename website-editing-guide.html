Pages

Our website is filled with pages. How do you view them, edit them, or add them?

In the left navbar, head to Pages -> All Pages. Some notable pages: our front page (named "Events"), our phone list (named "Phone List" by Tom Bousso), board pages, cheers, etc.

To add a new page, simply hit the "Add New" button on top, enter a title, and start adding content. If you know HTML, I'd consider making the page in "Text" mode, otherwise stick to "Visual" mode (the default one).

It's definitely a good idea to look at other similarly formatted pages to the one you want to create, and mimic the format found in those pages.

If you want the page to show up on the website, make sure to give it a "Parent" page (found in the right side of the editor, under "Page Attributes").

I'd highly recommend always previewing your edits before publishing them, which can be done by hitting the "Preview" button found in the top-right of the page.

If you want to make the page password protected (like this page), change the "Visibility" setting.

By the way, below your editor, you'll see a few sections. The only important section here is the "Editors" section. For example, if you only want the Aleph S'gan to be able to edit a page, then check off the two boxes in "Editors", and then find the aleph_sgan in the list of eligible users. You shouldn't normally need to mess with this, however.

As soon as you publish, find your page online by following the permalink on the page (which you can always customize). Move the page to trash if it's no longer necessary.

By the way, if you want your page to show up in the menu of the main website, you'll have to do this manually. This is pretty simple, just head to Appearance -> Menus from the left navbar. By the way, this is probably the only section of Appearance you will ever need to change. From there, find your page in the left "Pages" menu, and add it to the menu. You'll find it in the bottom of the "Menu Structure", and from there you can just drag it to the correct position. Be sure to "Save Menu" when you're done.

Changing the Front Page

One of the most common requests the Machshevim get from random Alephs is to add things to or make changes to the front page. How do you do that?

If you remember from the previous section, the front page is a page called "Events". The page editor is shown below.

Text view is something that for the most part, you can get by fine without touching it. A Page wants me to Edit it in Text mode

So you want to edit some page (maybe even this page), and it's asking you to edit it as text. You go over to the Text mode, and it's super confusing. What do you do?

Well, hate to break it to you, but this stuff is pretty confusing. Don't fret though, I'll guide you through it all.

First of all, if you head over to the Text mode, you'll see a comment with some form of an explanation. In that comment there's a link. Let's follow that link and see where it leads us (just copy paste it into the url of a new tab).

Ooh, it seems to lead to some website containing the exact same content. That's promising. But how to edit it?

Disclaimer: If you already know how to deal with GitHub and already have an account, I would just add your own account as a contributor to this repository, make your changes from there, and then copy paste your changes back into the wordpress editor in Text mode. In any case, you need to read a bit more to see how to get access to this content.

Boy, it would be sweet if I had some account in this website to edit it. What's that? I already have one? Sweet! But really, head over to the "Sign in" link on the top right navbar.

Now to log in, enter your username (ramonaza) and password. What's your password? Well, you can find out here. If the page looks blank, that probably means you're not an admin on the website, or you're not logged in.

Note that the user interface of GitHub can change over time, so the following instructions might not be 100% accurate, but you should still be able to follow them, so do the best you can.

Now that you're logged in, you will see some icons to the right of where it says "Raw", "Blame", and "History". Click on the first one (the pencil) to edit the file.

At this point, you can edit the file in one of a thousand ways. Probably the simplest way for you would be to edit on github, copy paste the code to the page on the website, see the preview of that page, and keep making changes until you're satisfied. Okay, so now what do you do when you're done editing? How do you save your changes?

If you scroll down, you'll see a section labeled: "Commit changes". Write a short message explaining the change you made. For most edits, you should also consider writing an extended description below. When you're done, git "Commit changes", and now it's official... on GitHub (please only do this when you finished making changes and you're sure they work).

You're still going to want to publish your changes to the website, though. So after you commit, you're going to want to hit the "Raw" button on GitHub. This will show you the plain text of the file. Now you can just copy paste all of it (you can use ctrl + A to select everything) into the website Text mode editor. Update your changes, and you're done!

Yippee!

Changing the Style for the Front Page Events

What do you do if you want to change the way events are displayed in the home page? Well, you've come to the right place.

First of all, you have to navigate to the right place. Head to the event settings page (mentioned in the last section). From there, go to the "Formatting" tab, and open up the "Events" section.

The sections in "Default event list format header" and "Default event list format" should look familiar to you if you already read the A Page wants me to Edit it in Text mode section. If you haven't already read it, read it fully now. You will need to follow its instructions in order to make changes to either of these two boxes.

If you want to make changes to the style of the events, you'll want to edit the first box ("Default event list format header"). Be sure to edit in GitHub, and then copy your changes back. This sheet is written using "css". If you don't know how to use it, look up tutorials online for "css".

If you want to change the layout, you'll want to edit the second box, labeled "Default event list format". Again, edit it on GitHub, and copy your changes back. This is written in HTML, find a quick tutorial online if this is new to you. In general though, this shouldn't be too hard to follow, and you should be able to make minor changes just fine.

When you're done making changes on GitHub, be sure to "commit" your changes, and copy paste them back to the website. Then, click the "Save Changes (All)" button that you'll see if you scroll down a bit.

Adding Event Attributes

Okay, the S'gan wants you to add an event attribute, and you have no clue how to do that. By the way, an event attribute is a thing like: "desc", "food-loc", "bring", "planner", etc. that you see when adding events (see this section for more information). So how is it done?

Disclaimer: Please avoid doing this, as it messes with the app, requiring it to take into account the new field ;( Only do the following if you really must.

It's pretty simple. If you head to the events settings page, in the "General" tab, under "General Options", you will be able to see a text area labeled "Event Attributes". This probably looks like exactly what you want. Add a new field corresponding to what you want to add (following the format of the others).

Afterwards, scroll down, and hit "Save Changes (All)". Now, head over to the "Formatting" tab, and open the "Events" section. You should go over the previous section first, or you're be very confused in the next few steps.

In the second text block, labeled "Default event list format", follow the link in that top comment (starting with github.com). This will take you to the GitHub page with that code.

To add your new attribute, you're probably going to want to copy one of those "div"'s, and modify it to fit your new attribute. For an example, view the third image below.

At this point, before committing your changes, you should copy paste your code into the "Default event list format" text box in the Event Manager settings, scroll down, and hit "Save Changes (All)"

To test if the code works, read the section explaining how to add an event. If you follow those instructions, you should now see a new attribute show up. Fill in some test value for it. For the "When" section, be sure you put a date in the future, or the event won't show up on the website, and make sure the start time is before the end time.

Hitting "Preview" wouldn't help, so just hit "Publish". When that's done, quickly go to the home page of ramon (ramonaza.org) and check if it's working.

If it works, great, delete your event as soon as possible (by moving it to trash), or change its visibility so that most people won't be able to see it. Keep reading on though!

If you can't see any event, that means you probably didn't set the correct dates in the "When" section of the previous page. If you just can't see the new attribute, be sure you gave it some value in the add event page, and be sure that you copy pasted your new content from GitHub into the "Default event list format header" text area and saved your work.

Now that it works, be sure to make your "commit" on the GitHub page, so that your edit will be saved permanently.

You're done! Go tell the S'gan that he can start using his new attribute as soon as he wants, and he could even edit pre-existing events and add in the new attribute to them!

Plugins

You may have noticed a Plugins section in the left navbar. Let's take a look at it (you can follow with the picture below if you aren't currently Machshev).

The first thing you'll notice is that some plugins have the words "Activate", "Edit", and "Delete", while others are highlighted and contain the words "Deactivate" and "Edit", amongst others. As you may have guessed, the unhighlighted ones (starting with "Activate") are not yet activated, meaning that they're not currently being used, while the highlighted ones (starting with "Deactivate") are currently being used.

You might be asking, why don't we just delete all the plugins that aren't being used? The answer is pretty complex. They might have once been used, but have since broken down, but the Machshev of the time thought to keep it there hoping that an update would eventually fix it. It might be something that a Machshev once wanted to implement but never got to it. Or it could simply be a Machshev asking the question, why remove it if it doesn't do any harm when deactivated?

In any case, feel free to delete unused plugins, but understand that they don't harm anything. However, be sure to always deactivate a plugin, check thoroughly if anything breaks, and then delete. Never jump to just deleting, because that could cause a lot of damage.

In order to understand more information about plugins, read on!

Plugins Overview

Woah, there are a lot of plugins, and you might not be sure what they all do. Before you go off deleting all of them, take a few minutes to be informed to be sure you don't go breaking anything bad.

Before we start, don't delete the "Events Manager" plugin, or the "Role Scoper" plugin. Deleting any of these two plugins will cause havoc, and break a lot of infrastructure we have in place. You might not see their effects at first, but if you look at what they do you'll be sure to find it. Also, never delete any activated plugin relating to galleries. These plugins store a lot of our media, meaning that if you remove them, you would potentially lose a lost of that chapter's history.

Now let's look at all the useful plugins and what they do:

  • Better WordPress reCAPTCHA: This plugin is being used to prevent a bot from creating a bunch of fake users for the website by adding a captcha to the registration page. This happened to us once, and never happened to us since this extension.
  • Capability Manager Enhanced: This plugin was used to create the S'gan's unique "Editor Plus Events" role (which allows the S'gan to edit its own pages and manage events, which is integral for the S'gan). Deleting the plugin won't remove or change the existing role, but would make it harder to modify it in the future.
  • Custom 404 Pro: We have a custom 404 page (which you see when going to a page that doesn't exist). That page is possible with this plugin (otherwise it would be a generic 404 page).
  • Easy Timer: I'm not entirely sure where this is used. We probably had a countdown timer at some point. This plugin is probably a good plugin to make that happen.
  • Edan is sexy: This plugin is renamed from "Pit Login Welcome". It's supposed to make a friendly welcome message for users when they log in.
  • Events Manager: Don't remove! This plugin handles all our event functionality that the S'gan constantly uses. This plugin is also super nice in the way that it allows us to create an RSS feed that our app can communicate with!
  • Facebook: Integrates posts with Facebook. Seems interesting, don't remember anyone ever using it, but hell, you can change that.
  • One of the Galleries: We have a few activated galleries, and they're all being used somewhere on the website. They also all suck, so if you find a better one, get the Historian Shotare to start using it. However, never remove them because they also store a bunch of the images—we would lose a ton of images if you remove any of these plugins.
  • Meta Slider: Adds picture slideshows in your posts.
  • Raw HTML: Allows some post to contain pure HTML in the Text editor without wordpress messing it up (it would still mess the post up if you try editing it in the visual mode).
  • Restrict Content: There is a page that contains information only for Administrators (passwords and stuff) that we don't want anyone else to be able to see. This plugin allows that. Without this plugin, that content becomes public.
  • Role Scoper: Allows easy modification of roles, and easy creation of new rules to give to users. This is used for the S'gan's role, as mentioned above. If you notice that the Editor Plus Events role weirdly has access to "activate plugins", that's not a bug. Without this access, S'ganim would not be able to modify or see event locations after making them for some weird reason.
  • RSVP: This plugin was once used to try to get an RSVP system working. I don't think we ever got it to work. You could try if you want.
  • TinyMCE Advanced: Adds a lot of nice features for the visual editor (in pages and such). This is pretty useful for people who use the visual editor, as some things are very annoying to do without this plugin.
  • WP Simple Pay (Lite Version): We tried getting a donate button on the website. I'm not sure if that worked. It might have, I never saw it though.

Updating Plugins

Sooner or later, you're bound to get some little icons close to the next navbar indicating that some plugins have updates available. This isn't really that scary, don't freak out. You just have to know how to deal with it.

There are really 2 ways. The first way, and probably easiest way, is just to never update them. And to be honest, there might never be a need to. However, sometimes updates contain fixes for bugs, or new features, and you might just want to always keep your plugins up to date. How do you do that?

There are really two ways to update plugins, and they're both essentially the same. The first is by just going to the plugins page from the left navbar (Plugins -> Installed Plugins). From here, you can find the plugins that contain updates. If the plugin isn't activated (it shows "Activate" under it instead of "Deactivate"), then updating it is pretty pointless, but also can't hurt anything. If if it's an activated plugin, meaning that it's probably in use, then it might be worthwhile to hit the "View version [version #] details" before updating in order to make sure it isn't going to ruin anything.

Once you're essentially satisfied with the change, you can update the plugin, or you can choose not to.

The second way to update plugins, is by hitting the little circular arrow button () on the top navbar. From there you can just see a list of plugins, and it is much easier to update them from here. However, in this page you can't see which of them are currently being used and which aren't.

I'd 100% recommend trying to use the website right after updating, especially if you didn't look at all the changelogs. If you broke something, you want to find out as soon as possible. What do you do if you just broke something? You can find out here.

I Updated Plugins and Something Broke

So maybe you didn't go over the changelog of all the plugins, and now something broke when you updated them. What do you do?

Go through the plugins you updated one by one, and if they are activated, try deactivating them. With one of the plugins, doing this should fix whatever was broken.

Now, if that plugin is a very important plugin, that's not good, so try looking up online, researching, getting help from Dan Appleman, do whatever you can to fix the issue.

If it's not really that important of a plugin, you can just leave it deactivated, fix anything that relied on it, and then go on with your day.

Good luck!

Removing Users

In this section, I'm not going to teach you how to remove users, as that should be self explanatory if you already got to this point in the guide. However, please read on.

If you look at the user list, you will see a list of users associated with board positions, but even more users who are Administrators, many of whom you've probably never heard of. And you may consider removing them. Here's why you shouldn't.

As you may have guessed, these past Administrators that you see were Machshevim at some point, just like you. In a normal company, employees usually stick around long enough to pass on all their knowledge. In Ramon, a lot of these Machshevim make a lot of changes, and then when 6 months pass, they're out.

What does this mean? When you edit pages, tools, etc., it will tell you who originally made that page, wrote that tool, or anything. If you're confused about something, the absolute best source of help is the original writer of that tool. I would constantly email or facebook message people who I saw who created specific things, and they'd just hop on and help me with whatever it is I requested. This was infinitely valuable.

Removing these users, removes your greatest support system.

And when you leave, be certain that the next Machshev is going to want your help, and perhaps the next Machshev, or some Machshev editing your content years later. These things happen.

Furthermore, there are some pretty cool people over there (and pretty smart). You guys might not know this, but the creator of the website is the now-alumnus Tom Bousso, and guess what, his account is still on the website. That's pretty sick!

I'm releasing this guide as an alumnus myself, as a member of the last class who was there when the new website was created. There's a lot of history there, and I think it's awesome.

Anyway, I hope that makes sense to you. Of course feel free to delete an account if someone made it by accident or anything, just think twice about deleting a user (or removing a plugin for that matter) if it's not causing any harm.

Thanks for reading!

Adding an Event

From the event page I had previously shown you how to get to, click on the "Add Event" button.

From here, things need to be done in a specific way. The name of the event will go in the title section, where it says "Enter title here".

For the location, enter it in "Location Name". First try searching for the location by name, e.g.: Ofek's Home, SPHDS (pick one with the address), Palo Alto JCC. Note that once you pick a location, you need to click the blue link right under it if you want to pick a new location. If you location doesn't already exist, then add a new location by going to Events -> Locations. You can see some more details on this in my adding a location section.

Note that nothing you put in the first big editor box, or the "Excerpt" box will actually do anything, so leave them blank. The place you care about is the "Attributes" section down below.

The most important attributes that you should always fill out are: "desc" -> the event description given to you by the planners, "bring" -> a list of items to bring (could be: money for food), and "planner" -> who planned the event (could be an Aleph name, or "The Chapter", or something else).

The other attributes aren't mandatory, so if you don't know them yet, you don't need to know them in order to publish the event. However, once you find them out, they're nice to have (parents bug the Metaksher about these things all the time). You can always edit this event after publishing it, add a few attributes, and then update the page.

Adding a Location

You're making a new event, but you can't find a location that you need. How do you make a new location? Or what if someone moved homes, and you have to update their address?

Luckily, this is a very simple task. Using the left navbar, simply head to Events -> Locations. Then click on the "Add Location" button on top. If instead you just want to change a location, then you can search for the locations in the top right.

This is pretty simple. Write the name of the location on top, where it says "Enter title here". Then, enter in as much information about the location in the "Where" section as you want. Just like in the events page, you can leave the main editor blank, as it won't be used. When you're done, you can just publish the event, and start using it in your events!

Editing an event can be done similarly, just by finding the location, editing it, and then Updating the location when you're done.

Board Pages

So the Aleph Machshev is bugging you to update your board page. How do you do that?

First of all, be sure to follow the instructions on the first things first section if you haven't already.

Now I'm assuming you're already logged in, and are at your Dashboard, or some random page.

You're going to want to find your board page. To do this, from the left navbar go to Pages -> All Pages. Unlike what happens when the Machshev does this, you should only see a few pages. You guessed it, these pages were made by (or for) previous people with your position.

To edit your board page, you're going to want to edit the one which says "Parent Page: Board Pages" on it. Make your changes to the page, and then hit the "Preview Changes" button on the top-right to be sure it looks nice. Once you're satisfied with your changes, hit the "Update" button, and your board page on the website should be instantly updated.

What if you want to edit some of the other pages you have access to? I'd consult the previous holders of your board position to ask about that, and I'd also definitely check out the Pages section of this guide.
